# Security Policy

## Reporting a vulnerability

This repository is a static content / atlas site (single-file HTML, hand-written
vanilla JavaScript, no external dependencies, no server-side code). If you
discover a security or privacy issue — for example an unintended data exposure,
a content-injection vector, or a broken-link/typosquat risk — please report it
responsibly.

- **Contact:** ehstand.schule@gmail.com
- **Subject:** `SECURITY — augmanitai-atlas`
- Please include a description, reproduction steps, and (if relevant) the affected
  file or URL.

## Disclosure expectations

- Please give a reasonable time to investigate and remediate before any public
  disclosure.
- Do not exploit the issue beyond what is necessary to demonstrate it.
- We will acknowledge your report and keep you informed of remediation progress.

## Scope

- In scope: this repository's published files and the GitHub Pages site it serves.
- Out of scope: third-party platforms linked from the content (ORCID, Wikidata,
  Zenodo, DataCite), and the canonical restricted-access research records.

## No secrets in this repository

This repository contains no credentials, API keys, or access tokens by design.
If you believe a secret has been committed, treat it as a vulnerability and
report it to the contact above immediately.
